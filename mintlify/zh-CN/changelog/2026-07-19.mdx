## 新特性

- **可选启用的 MCP 集成。** 通过单个命令，跨 Forge 配置的每个 AI 工具管理第三方 MCP 服务器（例如 `context7`）。在你运行 `forge integrations add <name> --yes` 之前，磁盘上不会写入任何内容。包含用于接管你已手动配置的服务器的 `--adopt`，以及非破坏性的 `remove`。参见 [`forge integrations`](/zh-CN/cli/config#forge-integrations)。
- **知识路由器（`forge know`）。** 通过一个命令在你的 ledger、atlas 和 substrate 之间路由问题。参见[记忆命令](/zh-CN/cli/memory)。
- **深度验证（`forge verify --deep`）。** 共识式验证，具有更严格的完成门槛。参见[质量命令](/zh-CN/cli/quality)。
- **预提交门（`forge precommit`）。** 在提交落地前运行验证清单。参见[质量命令](/zh-CN/cli/quality)。
- **Radar（`forge radar`）。** 暴露依赖风险与漂移。参见 [Radar 指南](/zh-CN/guides/radar-deps)。
- **Ledger 同步 + déjà。** 通过 `forge deja` 在多台机器间保持 ledger 一致，并检测重复工作。参见[记忆命令](/zh-CN/cli/memory)。
- **Dash v2 和 `forge report`。** 焕新的本地仪表盘，以及可分享的静态 HTML 报告，包含 ledger、指标和影响范围。参见[配置命令](/zh-CN/cli/config#forge-report)。
- **`forge tools` + 自动 gitignore。** 选择你的主要 AI 编码工具，Forge 会写入相匹配的 `.gitignore` 条目，以便生成的配置和 `.forge/` 产物被正确忽略。参见[配置命令](/zh-CN/cli/config#forge-tools)。
- **`forge uicheck interact`。** 用于 UI 验证的全新 Playwright 驱动交互循环。参见[质量命令](/zh-CN/cli/quality)。
- **OpenAI + Gemini 供应商。** 自动检测以及 OpenAI 兼容调用路径 — 使用 `forge config add` 进行配置。参见[配置命令](/zh-CN/cli/config#forge-config)。
- **自定义网关模型映射。** 将层级名称重新映射到自定义网关的真实模型 ID，使路由能够与任何 OpenAI 兼容端点协同工作。参见[模型路由](/zh-CN/concepts/model-routing)。

## 更新

- **基于结果校准的路由。** 模型路由现在使用根据真实结果而非默认值校准的可测量升级门。参见[模型路由](/zh-CN/concepts/model-routing)。
- **v0.20 上手体验优化。** 每个命令的帮助输出、用于自动修复常见配置问题的 `forge doctor --fix`、首次运行的零配置连线，以及更清晰的仪表盘输出。参见[零配置上手指南](/zh-CN/guides/zero-config-onboarding)。
- **统一设计系统。** 在每个公开的 Forge 界面 — CLI、dash 和 report — 上呈现一致的外观和交互模式。
- **`context7` 现在为可选启用。** Forge 自身的 MCP 服务器仍是默认连线的唯一服务器；`context7` 及其他第三方服务器通过 `forge integrations` 按需安装。参见[简介](/zh-CN/introduction)。
- **Init 标志、profile 和 `FORGE_GUARD_STRICT`。** 新记录的 init 标志、profile 选择以及严格守护模式。参见[配置命令](/zh-CN/cli/config)参考。
- **旧存储已退役。** 使用 `FORGE_LEDGER_ONLY`，旧的重复存储被移除，改为使用单一 ledger。

## 修复

- **信任缺口已闭合。** 针对 v0.22.0 发布的深度重新审计闭合了涵盖记忆、验证和路由的 24 个跨层信任问题。
- **审计整改。** 来自一次内部审计的 P0 正确性与安全修复，加上 P1 产品加固。
- **正确性缺陷。** 七个修复，覆盖 `deja` 底限逻辑、worktree 加固、ledger 同步、cortex 指标、verify 基线、radar 使用和 pre-commit 解析。
- **Settings 模板清理。** 从默认 settings 模板中移除了无效的 curl-pipe 拒绝规则。
