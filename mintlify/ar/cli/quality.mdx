مجموعة Quality هي سطح التحقق والسلامة. راجع
[بوّابات التحقق](/ar/concepts/verification-gates) لكيفية تركيبها.

## `forge verify`

بوّابة تحقق مستقلة — اختبارات + رمز مُهلوَس + مصدرية.

```bash
forge verify
forge verify --deep       # multi-lens consensus (v0.19+)
```

## `forge scan`

بوّابة المهارات — فحص مهارة أو خادم MCP بحثًا عن الحقن / RCE / التسريب قبل التثبيت.

```bash
forge scan <path-to-SKILL.md-or-.mcp.json>
```

## `forge spec`

المواصفة كعقد — init (OpenSpec)، وقفل، وفحص الانحراف.

```bash
forge spec init
forge spec lock
forge spec check
```

## `forge taste`

تمكين أداة ذوق واحدة لواجهة المستخدم لهذا المستودع (بدون وسيط يعرض الخيارات).

```bash
forge taste                # list the profiles
forge taste <profile>      # brutalist · corporate · editorial · minimalist · playful
```

يكتب `DESIGN.md` ويُعدّل عتبات بوّابة `uicheck design`.

## `forge uicheck`

فحوصات حتمية لواجهة المستخدم.

```bash
forge uicheck contrast <fg> <bg>     # WCAG contrast ratio
forge uicheck fingerprint <file...>  # deterministic design fingerprint
forge uicheck design <file...>       # slop-distance + conformance gate
forge uicheck visual <file-or-url>   # Playwright-rendered check (opt-in tier)
```

## `forge harden`

توصيل ضوابط الأمان — gitleaks قبل الالتزام + إعدادات الصندوق الرملي.

```bash
forge harden
```

## `forge precommit` <sub>v0.19+</sub>

بوّابة على مستوى الالتزام — تُشغّل الحد الأدنى للتحقق عند وقت الالتزام، فيُلتقط
العمل الجزئي أو غير المُتحقَّق منه قبل أن يستقر.

```bash
forge precommit
```

## `forge radar` <sub>v0.19+</sub>

حلقات حداثة التبعيات — تُجمِّع تبعيات المشروع بحسب مدى حداثتها، ليظهر ما هو بائت أو
منحرف قبل أن يعضّ.

```bash
forge radar
```

راجع دليل [إبقاء التبعيات محدَّثة](/ar/guides/radar-deps) لكيفية قراءة الحلقات.
