The root cause

One root. Two faces.

Every headline failure — breaking callers, hallucinating APIs, re-reading the same files, repeating a corrected mistake — is a symptom of one thing: the model is stateless and eats flat text. It carries no model of your code and no memory of what it already tried.

FACE A

No world-model

It doesn't hold a live map of your code, so it can't reason about what a change breaks.

“If I change X, what else fails?”

FACE B

No memory, no learning

Each session re-derives understanding from zero and repeats the same mistakes.

“What did I already learn here?”

How it works

A cognitive substrate — not another model.

Cortex doesn't train a new LLM. It wraps the one you use in three non-transformer layers that give it the memory and self-correction a predictor can't have on its own.

Perception

World-model

An always-current code graph: what exists, and what a change to it will impact — the "if I edit this, these 12 callers break" it never had.

Hippocampus

Self-correcting memory

Detects a genuine mistake, distills a structured lesson, and re-confirms it against fresh outcomes. Following a lesson never inflates its own confidence — so a wrong one decays out instead of ossifying.

Foresight

Local error predictor

Learns from this repo's own correction history to flag a risky edit before you make it — and it's honest: a hard kill-switch keeps it on a simple heuristic unless the learned model measurably beats it.

The honest part

It won't nag. It can't ossify.

Memory that acts on false confidence is worse than none. Cortex is engineered so a wrong lesson dies faster than a right one grows — and a green build always wins.

Injection is never confirmation
Only independent outcomes — tests, builds, a human — move a lesson's confidence. It can't grade its own homework.
A human reversal always wins
A mistake is defined by a bad outcome, not by "differs from a lesson." Change your mind and the lesson retires — it never fights you.
Advisory, never blocking
Cortex can never block or break a tool call. The hooks are fail-safe by construction — worst case, they do nothing.
The predictor can't become vaporware
It's held to an AUC-PR gate on future-held-out data. If it doesn't beat the heuristic, it's abandoned. If features carry no signal, it disables itself.
